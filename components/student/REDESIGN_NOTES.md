# تحديث هوية مكونات الطلاب

الغرض: توحيد تصميم مكونات الطلاب مع الهوية البصرية الجديدة، مع الحفاظ على الوظائف الحالية وتحسين الاستجابة وإمكانية الوصول.

المكونات التي تم تحديثها:
- StudentTable
  - استخدام عناصر الإدخال الموحّدة Input وButton وفق الهوية.
  - إضافة حاوية فلاتر ذات حدود وزوايا دائرية ومتوافقة مع الألوان.
  - تحسين العناوين والطباعة لتتبع دليل الأنماط.
  - استجابة أفضل عبر flexWrap وتوزيع شبكي للفلاتر.
- StudentActions
  - استبدال قائمة MUI بقائمة موحّدة Menu المخصّصة للمشروع.
  - الحفاظ على الأيقونات داخل عناصر القائمة عبر تمريرها ضمن label.
  - تحسين إمكانية الوصول عبر aria-label لزر الإجراءات.
- AddStudentModal
  - استبدال البنية اليدوية بـ Modal وInput وLabel وCheckbox الموحّدة.
  - الحفاظ على تحميل المعلمين والتعامل مع الأخطاء كما هو.
  - تحسين تجربة المستخدم والهوية عبر ألوان وخطوط وهوامش متناسقة.
- StudentDetailsModal
  - استبدال البنية اليدوية بـ Modal وInput وLabel وCheckbox الموحّدة.
  - تحسين عرض سجل تسجيلات الدخول باستخدام Paper وعناوين واضحة.
  - إضافة عناصر تفاعلية قابلة للوصول (role، tabIndex، مفاتيح لوحة المفاتيح).
- StudentProgress
  - تخصيص LinearProgress ليتماشى مع ألوان الهوية وزوايا دائرية.
  - تحسين الطباعة والعناوين.

ملاحظات إمكانية الوصول:
- إضافة aria-label لعناصر التحكم الحرجة (الأزرار، الحقول، مؤشرات التقدم).
- دعم لوحة المفاتيح لزر إظهار/إخفاء التعقب.
- الحفاظ على اتجاه RTL في كل المكونات.

الوظائف:
- جميع الوظائف الأصلية بقيت بدون تغيير (البحث، الفلاتر، الإضافة، التعديل، الإجراءات، التحميل من الخادم).
- لم يتم تعديل واجهات الاستدعاء للخوادم أو أنواع البيانات.

الاستجابة:
- استخدام تخطيطات شبكية مرنة (grid) مع أعمدة متغيرة xs/md.
- استخدام flexWrap في شريط الفلاتر لتجنب الانكسار على الشاشات الصغيرة.

الاختبار والتحقق:
- تم تشغيل تشخيصات TypeScript للمكونات المعدّلة بدون أخطاء.
- يمكن تشغيل الخادم التطويري للتحقق البصري بعد إغلاق أي عملية تشغيل سابقة لـ Next.
